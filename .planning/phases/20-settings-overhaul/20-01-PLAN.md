# Plan 20-01: Settings Hub Restructure & User Profile Page

## Objective

Restructure the settings hub page into 4 HubSpot-style sections (Your Preferences, Account Management, Data Management, Tools) and build the user profile/preferences page at `/settings/profile`.

## What Already Exists

- `/settings/page.tsx` — settings hub with 3 categories (Organization, Security & Access, System)
- `/settings/organization/page.tsx` — org settings with General/Branding/Notifications/Security tabs
- `/profile/page.tsx` — redirects to `/settings`
- User API service with `getMe()`, `updateUser()` endpoints
- Auth context with current user info

## Tasks

### Task 1: Restructure Settings Hub Page

**File:** `apps/frontend/src/app/(authenticated)/settings/page.tsx`

Replace the current 3-category layout with 4 HubSpot-style sections:

1. **Your Preferences** — Profile, Notifications, Task defaults
2. **Account Management** — Account Defaults, Users & Teams, Audit Log, Integrations, Approvals, AI Settings
3. **Data Management** — Properties, Objects
4. **Tools** — Workflows, Import/Export

Each item: icon + title + description + chevron link. Keep the existing card-based layout but reorganize.

### Task 2: Build User Profile Page

**File:** `apps/frontend/src/app/(authenticated)/settings/profile/page.tsx`

Create the "Your Preferences > General" page with tabs:

- **Profile** tab: Edit display name, email, phone, avatar, timezone, language
- **Security** tab: Change password (form with current/new/confirm), enable/disable MFA, active sessions
- **Task Defaults** tab: Default task reminders, preferred assignment mode

Use existing `usersApi.getMe()` and `usersApi.updateUser()` endpoints.

### Task 3: Update Profile Redirect

**File:** `apps/frontend/src/app/(authenticated)/profile/page.tsx`

Change redirect from `/settings` to `/settings/profile`.

### Task 4: Update Navigation

**File:** `apps/frontend/src/lib/navigation.ts`

Update the admin items to include new settings sub-pages:

- Keep Workflows, Users, Audit Log, Integrations
- Add "Properties" link to `/settings/properties`
- Add "AI Settings" link to `/settings/ai`

## Success Criteria

- [ ] Settings hub shows 4 sections with correct links
- [ ] `/settings/profile` renders with Profile/Security/Task Defaults tabs
- [ ] Profile tab loads current user data and allows editing
- [ ] Security tab shows password change and MFA toggle
- [ ] `/profile` redirects to `/settings/profile`
- [ ] TypeScript compiles without errors
